1 Levels

1.1

Gamer: As a gamer I want all the elements such as soundtrack, visuals, mechanics, etc. to incorporate the style and story of the game in order to make the game very immersive, fun, and puzzling and make the player want to continue playing. 

Elaboration: Users should be intrigued by the gameplay and storyline enough to continue playing and reach the end of the game. If captivating enough, some players would even replay the game after beating it once. 

Constraints:Developing a captivating story with complimenting visuals, soundtrack, and mechanics in the given time scope is a bit of a challenge. Also group size is a constraint as we must assign people to work on other areas such as art, soundtrack, programming, level/puzzle design; with larger groups we could focus on every aspect a gamer wants.

Effort Estimation: ~70 hours

Acceptance Test: Test the game with about 20-30 gamers, afterwards ask them to fill a quick survey going over the game as a whole:

Was it fun? Was it challenging? Would you play it again? Was it boring? Was it too easy? Rate the art, music, mechanics, storyline, etc.

 

1.2

Noob: As a noob I want to have an in-game tutorial so that I can easily understand how to play the game.

Elaboration: Users should be able to learn how to play the game, so an in-game tutorial is necessary to ensure that new players understand what to do.

Constraints: none

Effort Estimation: 3 person hours

Acceptance Test: Do a run-through of the tutorial on the demo to ensure it is working properly.

 

1.3

SpeedRunner: As a speedrunner, I want a linear experience that remains predictable and unchanged every time I play the game so that I do not have to worry about random events ruining my speedruns.

Elaboration: The user wishes for an unbroken game. This means all potential bugs, softlocks, and random clipping errors must be patched.

Constraints: None.

Effort Estimation: 10 person-hours.

Acceptance Test: Complete the entire game while testing all boundaries, UI navigation, and level manipulation to check for clipping, bugs, and softlocks respectively.

 

1.4

Critic: As a critic I want to complete challenging tasks in the game so that I can find the game more engaging and worthy of giving a higher review.

Elaboration: The user wants to have a game with complex and engaging puzzles. Making our puzzles more complex will take more time in both design and implementation.

Constraints: Puzzles must fit within the allowed mechanics and capabilities of the game.

Effort Estimation: 10 person hours

Acceptance Test: Complete a run-through of the puzzles with focus groups to determine

if they find the puzzles sufficiently challenging.

 

1.5

Prof. Z: As a professor I/we want to see challenging yet well done designs so thatI can see the creators have a good understanding of Unity and learn how to collaborate with their teammates .

Elaboration: The user wants to see good design. The game should look nice, it should play nice, and it should be interesting. It should be well polished, and it should look like a group effort, no one person coding everything.

Constraints: The game does have to be simple enough for our beginner Unity team. We have to use git.

Effort Estimation: 24 hours (3 hours a week for the rest of the semester)

Acceptance Test: Game will be demonstrated to class. Professor will provide a grade.

2 music

2.1

Musician: As a musician, I want to explore and compose the OST of the game through its gameplay mechanics so that I can experience the feeling of placing my own personal touch on the game itself.

Elaboration: The user wants to feel like he/she is part of the game development process (specifically, the score). Allowing the user to play each instrument line of a simple, catchy OST in game may grant a more engaging experience.

Constraints: OST must fit the city theme of the game, plus must be simple enough to be “played” on in-game instruments.

Effort Estimation: 15 person-hours.

Acceptance Test: Allow several test subjects to play the game through and record their remarks on the simplicity and “catchiness” of the OST.

3 visuals

3.1 

Artist: As an artist, I want to see an aesthetically pleasing setting and interface so that I feel welcome and comfortable when visually exploring the game’s world. 

Elaboration: Oftentimes, the visuals and graphics are what appear most prevalently in people’s minds when they think about a game. By having a likeable art direction and easy to use interface, players will easily be able to navigate the game and commit its larger parts to memory. 

Constraints: Art takes time, even something as simple as 8-bit designs can take a while to put together, particularly for backgrounds and items. Even if the game is 100% functional, there’s no guarantee the artistic desires have enough time to be fulfilled all the way. 

Effort Estimation: 30+ person hours

Acceptance Test: Let players get accustomed to the UI, seeing if they stumble to find where certain buttons and things are. Then, ask them if they enjoyed the visuals after having them play it for a bit. 

4 User Interface

4.1

Regular player: As a regular player, I want the game to have a minimap so that I donâ€™t get lost or confused in the large city setting of the game.

Elaboration: The area of the game is going to be big and would be confusing to navigate without a map. The city has 8 different accessible buildings as well as barriers blocking certain streets and underground tunnels to travel between certain buildings. To make this easier to navigate for the user, the team will include a minimap with icons for the buildings, barriers, and player sprite.

Constraints: The map must accurately reflect the actual city and have the distances between locations correctly scaled.

Effort Estimation: 5 person-hours

Acceptance Test: Navigate the entire map with the player sprite to ensure that the icons are properly distanced on the map and that the player icon moves as it should.

4.2

The Busy Person: As The Busy Person I want to be able to save the game so that I can take breaks between playing it.

Elaboration: The game pause menu should include a way to save and exit the game, and the menu should have a way to start the game at that checkpoint.

Constraints: Unity probably has an easy way to do this, we just have to figure out what it is.

Effort Estimation: 5 person hours

Acceptance Test: The game should be able to be saved at most points in the game.

4.3

Forgetful: As a forgetful gamer, I want an inventory system to keep track of all of the objects (instruments and sheet music) that I collect as I progress through the game.

Elaboration: The game pause menu will display a type of shelf with darkened areas around the objects that has not yet been collected. Once the object is collected, it will appear in the pause menu.

Constraints: None.

Effort Estimation: 3 hours.

Acceptance Test: The inventory will be empty at the start and must hold each object and the object is collected in game.
